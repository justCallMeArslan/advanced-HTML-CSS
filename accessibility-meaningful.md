Meaningful links:

There are three rules for links to be meaningful:

1. text content of the <a> should indicate where link redirects (~100 characters).
2. if a link would open or download file , let user know what kind of file and its size.
   (e.g <a href='...'>2021 Sign Up Statistics (PDF, 1MB)</a>)
3. if link opens new tab, indicate it to user (e.g <a href='...'>GitHub (opens in new tab)</a>)

Meaningful image alts :

1. be accurate and equivalent in representing content and function.
2. be succinct. Content (if any) and function (if any) should be presented as succinctly as possible, without sacrificing accuracy. Typically, only a few words are necessary, though rarely a short sentence or two may be appropriate.
3. not be redundant or provide the same information as text near the image.
4. not include phrases like "image of ..." or "graphic of ...", etc. This would be redundant since screen readers already announce "graphic" along with the alt text. If the fact that an image is a photograph or illustration, etc. is important content, it may be useful to include this in alternative text.

for additional dtails : https://webaim.org/techniques/alttext/



Meaningful forms: 

1. providing all necessary instructions and cues.
2. associating form controls with a text label by using the label element.
3. implementing fieldset and legend elements to associate groups of checkboxes and radio buttons.
4. ensuring that the reading and navigation order is logical.
5. ensuring that the form can be completed and submitted using a keyboard.
6. testing that the form controls, labels, and functionality are understandable and usable.
7. alert the user of any validation errors in an apparent and accessible manner with informative feedback messages.
8. allow the user to easily access the form controls that need to be modified.
9. allow resubmission and revalidation of the form information.
