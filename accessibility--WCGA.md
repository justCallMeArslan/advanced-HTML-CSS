The four principles (POUR) :

1. Persceivable - user must be able to percieve information or user interfaces
   being presented. (e.g light text on light bg-color can be a problem)
2. Operable - users must be able to operate any user interfaces or navigation,
   and interfaces cannot require an interaction which user cannot perform. (e.g
   nav bar which can only be triggered by mouse hovering over it).
3. Understandable - users must be able understand any information or UI that is
   presented to them. (e.g error 404 or 113 means nothing, unless detailed and
   understanbdable information provided describing what that error stands for)
4. Robust - content must be accessible by current and advanced assistive tech and
   other user agents.

There are WCAG ating system in place whhich is :

- level A or essential support (minimum level of conformance)
- level AA or ideal support (golden median)
- level AAA or specialized support (difficult to maintain as some features can be
  impossible to adapt; but some critiical parts (payment e.g) shlould be in full
  compliance)
